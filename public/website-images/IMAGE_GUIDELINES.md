# Image Optimization Guidelines

## Overview
This document outlines the image optimization strategy for the A1 Aluminium, Glass & Netting Solutions website.

## Image Formats
- **Primary Format**: WebP (automatically generated by Next.js)
- **Fallback Format**: AVIF (for modern browsers)
- **Source Format**: JPG/PNG (original uploads)

## Image Sizes and Dimensions

### Service Images
- **Category Images**: 800x600px (4:3 ratio)
- **Service Detail Images**: 1200x900px (4:3 ratio)
- **Thumbnails**: 200x150px (4:3 ratio)

### Blog Images
- **Featured Images**: 1200x630px (OG image ratio)
- **Thumbnail Images**: 800x600px (4:3 ratio)

### Social Media Images
- **Open Graph**: 1200x630px
- **Twitter Card**: 1200x630px

### Logo and Branding
- **Logo**: 200x200px (square)
- **Favicon**: 32x32px, 16x16px

## Quality Settings
- **Hero Images**: quality={90} (high quality for above-fold content)
- **Featured Images**: quality={90} (blog posts, service details)
- **Card Images**: quality={85} (service cards, blog cards)
- **Thumbnails**: quality={75} (small preview images)

## Lazy Loading Strategy
- **Above the Fold**: Use `priority` prop for hero images and first visible images
- **Below the Fold**: Default lazy loading (automatic with Next.js Image)
- **Carousels**: Load first image with priority, lazy load others

## Alt Text Best Practices
1. **Be Descriptive**: Include service name, location, and context
2. **Include Keywords**: Naturally incorporate SEO keywords
3. **Avoid Redundancy**: Don't start with "Image of" or "Picture of"
4. **Be Specific**: "Aluminium windows installation in Mumbai apartment" vs "Windows"

### Examples
- ✅ Good: "Modern glass partition installation in Bandra office space"
- ❌ Bad: "Glass partition"
- ✅ Good: "Safety nets protecting balcony in high-rise Mumbai building"
- ❌ Bad: "Safety nets image"

## Sizes Attribute
Use responsive sizes to serve appropriate image sizes:

```tsx
// Full width on mobile, half width on desktop
sizes="(max-width: 768px) 100vw, 50vw"

// Three columns on desktop
sizes="(max-width: 768px) 100vw, (max-width: 1024px) 50vw, 33vw"

// Fixed width
sizes="(max-width: 1024px) 100vw, 1024px"
```

## Compression Guidelines
Before uploading images:
1. Use tools like TinyPNG, ImageOptim, or Squoosh
2. Target 70-80% quality for JPG
3. Remove EXIF data
4. Resize to maximum required dimensions
5. Convert to WebP when possible

## Performance Targets
- **Largest Contentful Paint (LCP)**: < 2.5s
- **Cumulative Layout Shift (CLS)**: < 0.1
- **Image Load Time**: < 1s on 3G

## Next.js Image Configuration
See `next.config.ts` for current configuration:
- Formats: WebP, AVIF
- Device sizes: 640, 750, 828, 1080, 1200, 1920, 2048, 3840
- Image sizes: 16, 32, 48, 64, 96, 128, 256, 384
- Cache TTL: 60 seconds

## Placeholder Images
Current placeholder images are SVG-based for development. Replace with actual images:
1. Take high-quality photos of completed projects
2. Ensure proper lighting and composition
3. Get customer permission for project photos
4. Include variety of angles and contexts
5. Show before/after where applicable

## Accessibility
- Always include descriptive alt text
- Use empty alt="" for decorative images
- Ensure text overlays have sufficient contrast
- Don't rely on images alone to convey information

## SEO Considerations
- Use descriptive file names: `aluminium-windows-andheri.jpg` vs `IMG_1234.jpg`
- Include location keywords in alt text
- Use structured data for images (ImageObject schema)
- Optimize for Google Image Search
- Include captions where appropriate

## Testing
Regularly test image performance:
1. Run Lighthouse audits
2. Check Core Web Vitals
3. Test on various devices and network speeds
4. Verify lazy loading behavior
5. Check image quality on different screens

## Maintenance
- Review and update images quarterly
- Remove unused images
- Update placeholder images with real photos
- Monitor image CDN performance
- Keep track of image sizes and optimize as needed
